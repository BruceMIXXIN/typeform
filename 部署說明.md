# Typeform Exporter — Railway 部署說明

## 需要部署的檔案（共 4 個）

```
app.py          ← 主程式
index.html      ← 前端介面
requirements.txt
Procfile
```

---

## 部署步驟

### 1. 建立 GitHub Repo，推上這 4 個檔案

```bash
git init
git add app.py index.html requirements.txt Procfile
git commit -m "init"
git remote add origin https://github.com/your-org/typeform-exporter.git
git push -u origin main
```

### 2. Railway 建立新專案

1. 前往 [railway.app](https://railway.app)
2. New Project → Deploy from GitHub repo
3. 選擇剛剛推上去的 repo
4. 等待自動部署

### 3. 設定環境變數

Railway Dashboard → 專案 → Variables，新增以下 4 個：

| 變數名稱 | 說明 |
|---|---|
| `TYPEFORM_TOKEN` | Typeform API Token |
| `GOOGLE_CREDENTIALS` | Google Service Account JSON（見下方說明）|
| `GDRIVE_FOLDER_ID` | Google Drive 目標資料夾 ID |
| `API_SECRET` | 自訂保護用 Token（可隨意設一組字串）|

#### GOOGLE_CREDENTIALS 怎麼填

把 `credentials.json` 的內容**整個壓成一行**貼進去：

```bash
# Mac Terminal 執行，會自動複製到剪貼簿
cat credentials.json | tr -d '\n' | pbcopy
```

#### 各變數的值在哪裡找

- `TYPEFORM_TOKEN`：已內建在原本的 n8n workflow（`tfp_B69b...`）
- `GOOGLE_CREDENTIALS`：從 Google Cloud Console 下載 Service Account JSON
- `GDRIVE_FOLDER_ID`：`1dinTp_CD7ptlfHMbb6yhUEqOrE3VnBaa`（原本的資料夾）
- `API_SECRET`：自己設，例如 `mixxin-2024`

---

## 部署完成後

打開 Railway 給的 URL（例如 `https://typeform-exporter.railway.app`）就是前端介面。

使用方式：
1. 貼上 Typeform Form ID
2. 點「API 設定」填入 Railway URL 和 API Secret
3. 按「執行」
4. 完成後點「↗ 開啟 Google Sheet」

---

## 確認部署成功

打開這個網址應該回傳 `{"status":"ok"}`：

```
https://your-app.railway.app/health
```
